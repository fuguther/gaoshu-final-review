# 高数期末复习站 — PRD（草案 v0.1 · 待确认）

> 写法约定：`[实测]` = 本次看代码确认的事实；`[待确认]` = 需要你拍板的需求。
> 目的：先把"做好"是什么对齐清楚，再让 Kimi 动手（Opus 审查）。

## 1. 背景与目的
- `[实测]` 一个**静态数学复习站**（纯 HTML/CSS/JS + MathJax，无构建）。覆盖高数下：第9章后几节（线/面积分）、第10章（级数）、第11章（常微分方程）+ 往年真题 + 技巧锦囊。
- `[待确认]` **目的三问**：① 给谁用？（你自己期末复习 / 也分享给同学）② 考试/deadline 什么时候？（决定"求快"还是"求全"）③ 怎样算成功？（建议口径："复习时点开任意一页，都能照标准步骤把那类题做对"）

## 2. 现状（实测）
- 外壳成型：全局导航/章节下拉、设计系统 `css/design-system.css`、MathJax、暗色切换、阅读进度条、回到顶部。
- **硬伤**：
  - `[实测]` **坏链**：首页链到 `tips/integrating-factor.html`、`tips/power-series-models.html`，但 `tips/` 下只有 `index.html`，这两个文件不存在 → 404。
  - `[实测]` **骨架/薄页**（有外壳+步骤标题，但缺真正的例题与完整解答）：`9.4-big-three(-problems)`、`10.3-fourier(-problems)`、`10.2-power-problems`、`11.1-first-order-problems`、`11.2-second-order-problems`、`tips/index`、`exams/index`。
  - `[实测]` 质量落差：满页约 200–400 行（如 `9.3-surface-problems` 381 行），骨架页仅 25–80 行。
- `[实测]` 素材在手：父目录参考 PDF（积分因子技巧 / 巧算10.3.14 / 复习笔记…）、`_archive`（旧版习题详解 HTML/JSON）、`高数B`（2021–2023 往年卷+官方/个人解答）。

## 3. "做好"的定义（候选目标，待你排序/勾选）
- **G1 内容补全**：骨架/薄页填到满页深度（每个题型 ≥1 道"考场标准答案"例题：题型识别→标准步骤→完整计算→答案），补齐坏链页。
- **G2 数学正确性**：逐页核对公式/步骤/答案（复习站出错=害人，Opus 重点把关）。
- **G3 前端打磨**：设计一致性、公式不溢出、移动端/响应式、暗色模式、导航交互细节。
- **G4 范围扩展** `[待确认]`：是否加 第8章 重积分？更多往年卷（高数B 2021–2023）？

## 4. 范围 / 非范围
- 暂定范围 = 首页已列：ch9.2–9.4 / ch10 / ch11 / exams / tips。
- `[待确认]` 是否纳入 第8章重积分 / 更多年份真题。
- 非范围（除非你要）：换技术栈（保持静态 HTML+MathJax）、上线部署、注册/后端。

## 5. 质量基线
- 题型页：每类题 ≥1 道完整例题，格式=题型识别→标准步骤→完整计算（不跳步）→答案，对齐现有满页（模板：`9.3-surface-problems`）。
- 公式全用 MathJax（行内 `\(..\)`、行间 `\[..\]`），渲染无报错、不溢出。
- 复用既有 `design-system.css` 类，不引入新风格；导航/页脚/页内跳转与现有页一致。
